# Haier Evo WM

Расширенная версия интеграции **Haier Evo** для Home Assistant с дополнительной поддержкой стиральных машин.

## О проекте

За основу взята оригинальная интеграция **haier_evo** от **and7ey**:

https://github.com/and7ey/haier_evo

Данный проект не является самостоятельной разработкой интеграции с нуля. Основная архитектура, работа с API и WebSocket принадлежат автору оригинального проекта.

В этой версии основной упор сделан на поддержку стиральных машин Haier Evo. Большинство изменений было выполнено с использованием ИИ и большого количества практического тестирования на реальном устройстве. Моих глубоких знаний протокола Haier или разработки подобных интеграций нет, поэтому любые замечания, предложения и помощь приветствуются.

Это не просто beta, это pre-alpha 😊

## Что добавлено

На текущий момент реализовано:

* Поддержка стиральных машин Haier Evo;
* Поддерживаемые сенсоры:

- Статус (выполняется, завершено, ожидание)
- Текущая программа (программа из общего списка программ) <sup>*требуется доработка</sup>
- Выбранная программа (программа, выбранная через панель стиральной машины)
- Описание программы (сенсор, созданный из нескольких параметров для быстрого понимания текущего режима)
- Длительность программы (сколько времени будет длиться стирка) <sup>*требует доработки, так как данные пересекаются с отложенным стартом, из-за чего совместное использование вызывает некорректное отображение</sup>
- Оставшееся время программы
- Этап программы (распределение белья, нагрев, стирка, полоскание, слив, отжим и т.д.)
- Прогресс программы (процент завершения) <sup>*требует корректировки, никогда не достигает 100%</sup>
- Температура (установленная температура стирки)
- Скорость отжима (установленные обороты)
- Уровень загрязнения (выбранный уровень загрязнения) <sup>*требует корректировки, присутствуют неизвестные значения</sup>
- Осталось полосканий (количество оставшихся полосканий)
- i-Time (дополнительное время стирки) <sup>*требует корректировки, иногда показывает нелогичные значения</sup>
- Отложенный старт (бинарный сенсор, показывает, включён или выключен отложенный старт)
- Режим ожидания (бинарный сенсор, отображает состояние при отключении дисплея)
- Удалённое управление (бинарный сенсор, показывает, включено или выключено удалённое управление)
- Функция пара (бинарный сенсор, показывает, включена или выключена функция пара)
- Антисминание (бинарный сенсор, показывает, включено или выключено антисминание)
- Детский замок (бинарный сенсор, показывает, включён или выключен детский замок)
- Блокировка дверцы (бинарный сенсор, показывает, включена или выключена блокировка дверцы)
- Дверца (бинарный сенсор, показывает, открыта или закрыта дверца)
- Звуковое оповещение (бинарный сенсор, показывает, включено или выключено звуковое оповещение)
- Расход воды (количество израсходованной воды)
- Потребление энергии (количество потреблённой энергии)
- Мощность (текущая потребляемая мощность) <sup>*требует корректировки, значения не соответствуют реальной работе стиральной машины</sup>
- Общее количество стирок (счётчик выполненных стирок)

Удалённое управление реализовать врядли получится, потому что команды принимаются только после нажатия кнопки на самой стиральной машинке.

## Проверенная модель

Интеграция протестирована на:

* **Haier HW80-B14979**
* **Haier HW70-BP12337**
* **Haier HW60-BP12929**

Работа с другими моделями возможна, но требует проверки.

## Если у кого то есть желание поковырять данные которые присылает машинка для добавления новых сенсоров:

Включаете какой либо режим или стирку, нажимате перезагрузить интеграцию, дальше проходите по адресу - http://ВашIPHA:8123/api/haier_evo, там будет большой файл, это и есть сыры данные, делаете опять какое то действие с машинкой, опять нажимаете перезагрузить интеграцию, а дальше сравниваете данные и делаете выводы о работе какого либо сенсора =)

## Установка

### 1. Загрузка интеграции

Скачайте последнюю версию интеграции на странице **Releases** этого репозитория.

Распакуйте архив.

---

### 2. Копирование файлов

Скопируйте папку:

```text
custom_components/haier_evo
```

в каталог Home Assistant:

```text
/config/custom_components/
```

В итоге структура должна выглядеть следующим образом:

```text
/config/
└── custom_components/
    └── haier_evo/
        ├── __init__.py
        ├── manifest.json
        ├── api.py
        ├── sensor.py
        └── ...
```

---

### 3. Перезапуск Home Assistant

После копирования файлов полностью перезапустите Home Assistant.

---

### 4. Добавление интеграции

1. Перейдите в **Настройки → Устройства и службы**.
2. Нажмите **Добавить интеграцию**.
3. Найдите **Haier Evo**.
4. Введите данные вашей учётной записи приложения **Haier Evo**.

После успешной авторизации интеграция автоматически обнаружит все устройства, привязанные к вашему аккаунту.

---

## Обновление

Для обновления интеграции достаточно заменить содержимое папки:

```text
/config/custom_components/haier_evo/
```

на файлы из новой версии и перезапустить Home Assistant.
